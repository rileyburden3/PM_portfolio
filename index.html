Cosmic Scale Explorer

Live

The problem: I wanted to deeply understand what building with Claude actually entails. So I built an interactive web app that lets anyone compare 70+ space objects at true to scale proportions, with AI generated fun facts for every pairing. Personal curiosity plus my 6 year old's interest in space equals a real product.

✨ See Live Demo
What I Built
  • 70+ space objects (asteroids, comets, planets, stars, exoplanets, black holes)
  • True scale visual comparisons with scrollable canvas for extreme size differences
  • Browsable grid with filters and live search
  • AI powered fun facts via Claude API (Claude Haiku)
  • Shareable URLs so anyone can send a comparison directly
  • Audio pronunciation for each object
How I Built It and What Surprised Me

I directed Claude end to end: defining scope, making architecture decisions, iterating on UX. Claude handled all code. What surprised me most was how collaborative and intelligent the process was. Not just code output, but scaffolding the entire product (GitHub repo, Vercel deployment, API security decisions). It felt like working with an engineer, except instantaneous and without competing priorities. What would have taken me a month took hours.

The PM Decisions That Mattered
  • Start simple: At its core, the app just needed to show sphere size differences, names, and simple UX. A one shot prompt got us 80% there.
  • Architecture for security: I decided against inline API keys in the browser, opting for a serverless function instead. A decision that required understanding the trade offs Claude explained.
  • User as customer: I scoped based on what my 6 year old actually needed, not feature creep. That constraint made the product stronger.
What I Would Do Differently

I skipped the formal structure. No vision doc, no PRD, no customer discovery beyond "what does my kid want?" Next time I'll document assumptions upfront and validate them systematically. This taught me that velocity without structure is seductive but risky.

What This Taught Me About Builder PM

I realized that my insecurity about not having a technical background was a limiting belief. I don't need to code to ship products. What I need is PM discipline (scoping, user centricity, architecture thinking) plus the ability to direct an AI tool thoughtfully. That's a skill I already have. The realization opened my eyes to what's actually possible for a builder PM.
